# APP_Idia_24ショートschedule

作成日: 2026-06-16  
種別: アプリアイディア資料  
格納先: 納品InBox

## 1. 企画概要

`APP_Idia_24ショートschedule` は、24時間を短い時間スケールに圧縮して扱う、TODO・カレンダー・カンバン・リマインダー・メール/チャット会議を統合した作業管理アプリの企画案です。

主な狙いは、プロジェクト管理が苦手なユーザーでも「今日の作業」「AI agent に依頼した作業」「納品された成果物」「次に判断すること」を一つの画面で把握できるようにすることです。

通常のダッシュボードや管理ツールは、項目が増えるほど迷走しやすくなります。このアプリでは、1日を短いゲーム内時間に変換し、専用時計とログを中心にして、作業の流れを見える化します。

## 2. 背景課題

- Google Keep のように、軽くメモしてリマインダーを付けたい。
- カレンダーのように、時間軸で予定を確認したい。
- カンバンのように、作業状態を `未着手 / 進行中 / 確認待ち / 完了` で見たい。
- Gmail やチャット会議のように、やり取りと成果物の文脈を残したい。
- AI agent に指示した内容、作業ログ、納品物が散らばりやすい。
- Dashboard や管理ツールを試しても、逆に管理すること自体が負担になりやすい。

## 3. 中心コンセプト

### 24時間を短縮して扱う専用時計

現実の1日を、そのまま24時間として見るのではなく、短い時間に圧縮して表示します。  
ユーザーは「今日をどう進めるか」を、普通のカレンダーよりも軽く、ゲームのターン管理に近い感覚で扱えます。

### AI agent 作業ログのシミュレーションゲーム化

AI agent を単なるチャット相手ではなく、作業者・受注者・納品者として扱います。  
ユーザーは agent に指示を出し、agent はログを残しながら作業し、成果物を納品します。  
この一連の流れを、シミュレーションゲームのように見える化します。

### TODO と納品物を同じ流れで扱う

TODO は単なるチェックリストではなく、依頼・作業・確認・納品の流れを持つ小さな案件として扱います。  
AI agent に投げたタスクも、自分でやるタスクも、同じボード上で管理します。

## 4. 時間圧縮モデル

この企画では、2種類の時間モデルを別々に考えます。

### A. 24時間を4時間に圧縮するモデル

名称案: `4H Day Clock`

- 24時間を240分として扱う。
- 現実の1時間を、アプリ内では10分として表示する。
- 比較的落ち着いた作業管理向け。
- 1日の計画、午前/午後/夜の区切り、まとまった作業ブロックを扱いやすい。
- 実用的な TODO 管理・カレンダー連携に向いている。

想定用途:

- 今日の作業計画
- 4時間集中セッション
- AI agent への複数依頼の進行確認
- リマインダー付きの短期プロジェクト管理

### B. 24時間を24分に圧縮するモデル

名称案: `24M Sprint Clock`

- 24時間を24分として扱う。
- 現実の1時間を、アプリ内では1分として表示する。
- 高速シミュレーション、ゲーム的な振り返り、作業ログ再生に向いている。
- 1日の流れを短時間で俯瞰できる。
- 実作業中というより、レビュー・訓練・シミュレーションに向いている。

想定用途:

- AI agent の作業ログ再生
- 今日の作業の振り返り
- 仮想プロジェクト運営ゲーム
- 依頼から納品までの流れを短時間で体験するチュートリアル

## 5. 主要機能案

### 1. 専用時計

- 4H Day Clock と 24M Sprint Clock を切り替え可能にする。
- 現実時間と圧縮時間を並べて表示する。
- 現在の時間帯に対応する TODO、予定、agent 作業を表示する。
- 作業の開始・中断・再開・完了を時計上に記録する。

### 2. タイムライン型カレンダー

- 1日を圧縮時間の帯として表示する。
- 予定、作業、リマインダー、会議、納品予定を並べる。
- Google カレンダー的な時間軸と、ゲーム内スケジュール表の中間を目指す。

### 3. Keep 型メモ + リマインダー

- 軽い思いつきメモをすぐ作れる。
- メモから TODO、agent 依頼、カレンダー予定へ変換できる。
- リマインダーを設定し、圧縮時計上にも表示する。

### 4. カンバン

基本レーン案:

- `Inbox`
- `Plan`
- `Doing`
- `Agent Working`
- `Review`
- `Delivered`
- `Archive`

AI agent に任せたものは `Agent Working` に入り、作業ログや中間成果物が自動で紐づきます。

### 5. Gmail / チャット会議風ログ

- タスクごとにスレッドを持つ。
- 指示、質問、判断、納品物を時系列で残す。
- 会議ログ、チャットログ、メール文面をまとめて検索できる。
- 成果物ファイルへのリンクをタスクに保持する。

### 6. AI agent 作業シミュレーション

- ユーザーは agent に作業依頼を出す。
- agent は作業ステータス、ログ、成果物を更新する。
- 作業が進む様子をゲーム内イベントとして表示する。
- 成功、差し戻し、追加質問、納品などをイベント化する。

## 6. 画面構成案

### メイン画面

- 左: 圧縮時計と今日の時間帯
- 中央: カンバンまたはタイムライン
- 右: 選択中タスクのチャット/ログ/納品物
- 下部: Inbox メモ入力バー

### Clock View

- 4H / 24M の切り替え
- 現在位置の表示
- 作業ブロックの色分け
- リマインダーのピン表示

### Agent Log View

- agent ごとの作業履歴
- 指示内容
- 中間ログ
- 納品物
- 次にユーザーが判断すること

### Delivery View

- 完了タスク一覧
- 納品物リンク
- 納品日時
- 確認ステータス
- 次の派生タスク

## 7. MVP 案

最初に作るなら、以下に絞るのがよいです。

1. 4H Day Clock
2. 手動登録の TODO
3. `Inbox / Doing / Review / Delivered` の簡易カンバン
4. タスクごとのログ欄
5. 納品物リンク欄
6. 24M Sprint Clock はログ再生モードとして後回し

最初から Gmail 連携や Google カレンダー連携まで入れると重くなるため、MVP では外部連携なしのローカル管理ツールとして設計します。

## 8. 発展案

- Google Calendar 連携
- Google Keep 風クイックメモ取り込み
- Gmail / Chat ログ取り込み
- AI agent への自動タスク発行
- 成果物フォルダとの連携
- 日次レビュー自動生成

- プロジェクト運営シミュレーションゲーム化

## 9. この企画の面白さ

このアイディアの核は、管理ツールを「管理するための表」ではなく、「作業が進む様子を見られる時計付きシミュレーション」に変えることです。

AI agent の作業は、裏側で進むと見えにくくなります。  
そこで、ログ・指示・納品をゲーム内のイベントとして扱うことで、ユーザーは「今なにが動いているか」「次に自分が判断することは何か」を直感的に把握できます。

24時間を4時間または24分に圧縮する発想は、普通の TODO アプリとの差別化ポイントになります。  
特に `4H Day Clock` は実用アプリ、`24M Sprint Clock` はレビュー/シミュレーション/ゲーム体験として分けると、企画として整理しやすいです。
